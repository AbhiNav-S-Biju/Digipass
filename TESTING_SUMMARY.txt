🔑 IMPORTANT NOTES
═══════════════════════════════════════════════════════════════════════════════

1. Bearer Token Format:
   Authorization: Bearer <JWT_TOKEN>
   (The "Bearer " prefix is REQUIRED)

2. Token Lifetime:
   Valid for 7 days from issue date
   After 7 days: Must login again

3. Password Security:
   Never sent back in API response
   Only hash is stored in database
   Always use HTTPS in production

4. Multiple Protected Routes:
   All protected routes use same middleware
   Can be scaled to any number of routes

5. Database Check:
   After testing, users are in database:
   SELECT * FROM users;
