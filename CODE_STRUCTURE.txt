PostgreSQL Table: digital_assets

Field           │ Type         │ Purpose
────────────────┼──────────────┼──────────────────────────────────
asset_id        │ SERIAL PK    │ Unique identifier (auto-increment)
user_id         │ INTEGER FK   │ Links to users table
asset_name      │ VARCHAR(255) │ User-friendly name (e.g., "Gmail")
asset_type      │ VARCHAR(50)  │ Category (email, bank, crypto, ...)
encrypted_data  │ TEXT         │ JSON with {encrypted, iv, authTag}
created_at      │ TIMESTAMP    │ Auto-set on insert
updated_at      │ TIMESTAMP    │ For future modifications

Cryptography Flow:
  ┌─────────────┐
  │ JSON object │
  └──────┬──────┘
         │
         ▼
  ┌──────────────────┐
  │ JSON.stringify() │
  └──────┬───────────┘
         │
         ▼
  ┌──────────────────────────┐
  │ Generate random IV (16B) │
  └──────┬───────────────────┘
         │
         ▼
  ┌──────────────────────────┐
  │ createCipheriv (AES-GCM) │
  └──────┬───────────────────┘
         │
         ▼
  ┌──────────────────┐
  │ cipher.update()  │
  ├──────────────────┤
  │ cipher.final()   │
  └──────┬───────────┘
         │
         ▼
  ┌──────────────────────┐
  │ getAuthTag()         │
  └──────┬───────────────┘
         │
         ▼
  ┌──────────────────────────────────────┐
  │ { encrypted, iv, authTag } → JSON    │
  │ Stored in encrypted_data column      │
  └──────────────────────────────────────┘

POST /api/assets (Add Asset)
────────────────────────────────────────────────────────────────────────────

REQUEST:
  Headers:
    Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
    Content-Type: application/json

  Body:
    {
      "asset_name": "Gmail Account",
      "asset_type": "email",
      "asset_data": {
        "username": "user@gmail.com",
        "password": "SecurePass123"
      }
    }

PROCESSING:
  1. authenticateToken middleware
     ├─ Extracts token from header
     ├─ Verifies JWT signature
     └─ Sets req.userId = 42

  2. addAsset controller
     ├─ Validates input fields
     ├─ Checks duplicate asset_name? (optional in future)
     ├─ Calls encryptData()
     │  └─ Returns {encrypted: "a1b2...", iv: "...", authTag: "..."}
     ├─ JSON.stringify encrypted result
     └─ INSERT INTO digital_assets

  3. Database
     └─ Returns: asset_id, asset_name, asset_type, created_at

RESPONSE (201 Created):
  {
    "success": true,
    "message": "Asset added successfully",
    "data": {
      "asset_id": 1,
      "asset_name": "Gmail Account",
      "asset_type": "email",
      "created_at": "2026-04-12T12:34:56.000Z"
    }
  }

────────────────────────────────────────────────────────────────────────────

GET /api/assets (List Assets)
────────────────────────────────────────────────────────────────────────────

REQUEST:
  Headers:
    Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

PROCESSING:
  1. authenticateToken middleware
     └─ Sets req.userId = 42

  2. getAssets controller
     └─ SELECT asset_id, asset_name, asset_type,  created_at
        FROM digital_assets
        WHERE user_id = 42
        ORDER BY created_at DESC

  3. Database
     └─ Returns array of assets (no encrypted_data)

RESPONSE (200 OK):
  {
    "success": true,
    "message": "Assets retrieved successfully",
    "data": [
      {
        "asset_id": 1,
        "asset_name": "Gmail Account",
        "asset_type": "email",
        "created_at": "2026-04-12T12:34:56.000Z"
      }
    ],
    "count": 1
  }

────────────────────────────────────────────────────────────────────────────

DELETE /api/assets/:id (Delete Asset)
────────────────────────────────────────────────────────────────────────────

REQUEST:
  URL: /api/assets/1
  Headers:
    Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

PROCESSING:
  1. authenticateToken middleware
     └─ Sets req.userId = 42

  2. deleteAsset controller
     ├─ SELECT FROM digital_assets
     │  WHERE asset_id = 1 AND user_id = 42
     ├─ If not found → 404 response
     ├─ If found:
     │  ├─ DELETE FROM digital_assets WHERE asset_id = 1
     │  └─ Return deleted asset info

RESPONSE (200 OK):
  {
    "success": true,
    "message": "Asset deleted successfully",
    "data": {
      "asset_id": 1,
      "asset_name": "Gmail Account"
    }
  }

Or (404 Not Found):
  {
    "success": false,
    "message": "Asset not found or unauthorized"
  }

                          SECURITY ARCHITECTURE
═══════════════════════════════════════════════════════════════════════════════

Encryption Layer:
┌─────────────────────────────────────────────┐
│ Plaintext Asset Data                        │
│ {username, password, credentials, ...}      │
└──────────────────┬──────────────────────────┘
                   │
                   ▼ encryptData()
┌─────────────────────────────────────────────┐
│ Ciphered + IV + AuthTag                     │
│ {encrypted: "...",                          │
│  iv: "...",                                 │
│  authTag: "..."}                            │
└──────────────────┬──────────────────────────┘
                   │
                   ▼ Stored in DB
┌─────────────────────────────────────────────┐
│ PostgreSQL encrypted_data Column            │
│ (as JSON string)                            │
└──────────────────┬──────────────────────────┘
                   │
                   ▼ Query returns NO plaintext
┌─────────────────────────────────────────────┐
│ API Response                                │
│ {asset_id, asset_name, asset_type, ...}    │
│ NO encrypted_data, NO plaintext             │
└─────────────────────────────────────────────┘

Access Control:
  Client
    │
    ├─ POST /api/assets
    │  └─ authenticateToken → req.userId set
    │     └─ addAsset() → uses req.userId to link to user
    │
    ├─ GET /api/assets
    │  └─ authenticateToken → req.userId set
    │     └─ getAssets() → WHERE user_id = $1
    │
    └─ DELETE /api/assets/:id
       └─ authenticateToken → req.userId set
          └─ deleteAsset() → verifies ownership before delete
