                          API ENDPOINT DETAILS
═══════════════════════════════════════════════════════════════════════════════

POST /api/assets
├─ Purpose: Add new digital asset
├─ Auth: Required (Bearer token)
├─ Body: {asset_name, asset_type, asset_data}
├─ Response: 201 (asset metadata)
└─ Encrypt: asset_data → {encrypted, iv, authTag}

GET /api/assets
├─ Purpose: Retrieve user's assets
├─ Auth: Required (Bearer token)
├─ Query: None
├─ Response: 200 (array of assets)
└─ Security: No encrypted_data in response

DELETE /api/assets/:id
├─ Purpose: Delete asset
├─ Auth: Required (Bearer token)
├─ Param: asset_id
├─ Response: 200 (deleted asset info)
├─ Check: Ownership verified before delete
└─ Result: 404 if unauthorized

═══════════════════════════════════════════════════════════════════════════════
                        ENCRYPTION PROCESS
═══════════════════════════════════════════════════════════════════════════════

1. Client sends plaintext:
   {
     "asset_name": "Gmail",
     "asset_type": "email",
     "asset_data": {
       "username": "user@gmail.com",
       "password": "MySecurePassword123"
     }
   }

2. Server encrypts asset_data:
   ├─ Generate random 16-byte IV
   ├─ Create cipher with AES-256-GCM
   ├─ Encrypt JSON string
   ├─ Get authentication tag
   └─ Result: {encrypted: "...", iv: "...", authTag: "..."}

3. Database stores encrypted JSON:
   {
     "asset_id": 1,
     "user_id": 42,
     "asset_name": "Gmail",
     "asset_type": "email",
     "encrypted_data": "{\"encrypted\":\"a1b2c3...\",\"iv\":\"...\",\"authTag\":\"...\"}"
   }

4. API response (no plaintext):
   {
     "success": true,
     "data": {
       "asset_id": 1,
       "asset_name": "Gmail",
       "asset_type": "email",
       "created_at": "2026-04-12T12:34:56.000Z"
     }
   }
